# QAI Institute

Trilingual (EN/KZ/RU) digital archive of Kazakh architectural identity.

**Stack:** Next.js 14 (App Router) · TypeScript · Tailwind CSS · Decap CMS · Cloudinary · Mapbox GL JS · Netlify

---

## Local Development

### Prerequisites

- Node.js 20+
- npm

### Setup

```bash
git clone <repo-url>
cd qai-institute
npm install
cp .env.example .env.local
# Fill in your API keys in .env.local
npm run dev
```

The site runs at [http://localhost:3000](http://localhost:3000).

### Environment Variables

| Variable | Description |
|---|---|
| `NEXT_PUBLIC_MAPBOX_TOKEN` | Mapbox public access token — get from [mapbox.com](https://mapbox.com) |
| `NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME` | Cloudinary cloud name — get from [cloudinary.com](https://cloudinary.com) |
| `ANTHROPIC_API_KEY` | Anthropic API key for news summarisation — get from [console.anthropic.com](https://console.anthropic.com) |
| `NEXT_PUBLIC_SITE_URL` | Full site URL (e.g. `https://qai-institute.netlify.app`) |

### CMS in Development

To test the CMS locally:

```bash
# In one terminal:
npm run dev

# In another terminal:
npx decap-server
```

Then visit [http://localhost:3000/admin/](http://localhost:3000/admin/).

## Netlify Deployment

### 1. Connect to Netlify

1. Push this repo to GitHub
2. Log in to [netlify.com](https://netlify.com) → **Add new site** → **Import an existing project**
3. Select your GitHub repo
4. Build settings are pre-configured in `netlify.toml`:
   - Build command: `npm run build`
   - Publish directory: `.next`

### 2. Set Environment Variables

In **Site settings → Environment variables**, add:
- `NEXT_PUBLIC_MAPBOX_TOKEN`
- `NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME`
- `ANTHROPIC_API_KEY`
- `NEXT_PUBLIC_SITE_URL` (your Netlify URL)

### 3. Enable Netlify Identity

1. **Site settings → Identity → Enable Identity**
2. Under **Registration**, set to **Invite only**
3. Under **Services → Git Gateway**, click **Enable Git Gateway**
4. Invite your editor email via **Identity → Invite users**

### 4. Access the CMS

Visit `https://your-site.netlify.app/admin/` and log in with your Netlify Identity credentials.

## Cloudinary Setup

1. Create a free account at [cloudinary.com](https://cloudinary.com)
2. Note your **Cloud Name** from the dashboard
3. In the CMS config (`public/admin/config.yml`), replace the `cloud_name` placeholder with your cloud name
4. Upload images via the Cloudinary dashboard or directly through the CMS media widget

Images are stored as Cloudinary **public IDs** in the content files (not full URLs). The site constructs the full URL using the `NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME` env var.

## Content Management

Content is managed through the Decap CMS at `/admin/`. All content is stored as Markdown files in the `content/` directory and committed to git automatically.

### Collections

| Section | Folder | Notes |
|---|---|---|
| Buildings | `content/buildings/` | Trilingual fields, coordinates, photos, conservation notes |
| Research | `content/research/` | Academic citations with multilingual abstracts |
| Gallery | `content/gallery/` | Cloudinary image IDs, captions, building references |
| History | `content/history/` | Timeline periods, ordered manually |
| Future | `content/future/` | Articles, projects, proposals, manifestos |
| News | `content/news/` | Auto-generated by scraper — read-only in CMS |
| About | `content/about.md` | Single file — mission, bio, contact |

### Adding a Building

1. Go to `/admin/` → **Buildings** → **New Building**
2. Fill in all trilingual fields (EN/KZ/RU)
3. Enter latitude/longitude (decimal degrees)
4. Upload photos via the Cloudinary widget
5. Save → the site rebuilds automatically on Netlify

## News Scraper

The scraper (`netlify/functions/scrape-news.ts`) runs daily at 06:00 UTC via Netlify Scheduled Functions. It:

1. Fetches RSS feeds from ArchDaily, Dezeen, The Architect's Newspaper, and vlast.kz
2. Filters articles for Kazakhstan/Central Asia relevance
3. Uses the Anthropic Claude API to generate 2–3 sentence summaries
4. Saves results as Markdown files in `content/news/`

To trigger the scraper manually:
```bash
curl -X POST https://your-site.netlify.app/.netlify/functions/scrape-news
```

## i18n

- Locale is detected from the URL prefix: `/en/`, `/kz/`, `/ru/`
- UI strings live in `messages/en.json`, `messages/kz.json`, `messages/ru.json`
- Content is trilingual per CMS field (e.g. `title_en`, `title_kz`, `title_ru`)
- Middleware (`src/middleware.ts`) handles locale routing via next-intl

## Swapping the Logo

The logo is referenced via a config constant:

```ts
// src/lib/config.ts
export const LOGO_PATH = "/logo.svg";
```

Replace `public/logo.svg` (or change the constant to point to `logo.png`) without any code changes.
